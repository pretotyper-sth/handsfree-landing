## 🛠 제작 프로세스

### Phase 1: 기본 구조 (30분)

```
1. 프로젝트 폴더 생성
2. index.html / styles.css / script.js 기본 구조
3. 핵심 섹션 배치
   - 헤더 (로고, 태그라인)
   - 히어로/지도 섹션
   - 옵션 선택 (사이즈, 시간, 플랜 등)
   - 요약 & CTA 버튼
   - 푸터
```

### Phase 2: 디자인 & UX (1시간)

```
1. 브랜드 컬러 적용
2. 모바일 최적화 (타겟이 주로 모바일)
3. 터치 피드백 & 마이크로 인터랙션
4. 폰트 선택 (Pretendard 기본)
```

### Phase 3: 프리토타이핑 로직 (30분)

```
1. CTA 버튼 클릭 시:
   - 1번째 클릭 → "일시적 오류" 모달
   - 2번째 클릭 → 503 에러 페이지
2. 실제 동작 없음 (결제, DB 없음)
```

### Phase 4: 그로스 해킹 설정 (20분)

```
1. Google Analytics 4 연동
2. 이벤트 트래킹 추가
   - page_view (페이지 접속)
   - scroll_depth (스크롤 깊이 25/50/75/100%)
   - button_click (버튼별 클릭)
   - reserve_click (CTA 클릭)
   - session_end (이탈 시 체류시간)
```

### Phase 5: 배포 (10분)

```
1. GitHub 저장소 생성 (Public)
2. git init → commit → push
3. Vercel 연동 (GitHub 로그인 → Import → Deploy)
4. 커스텀 도메인 설정 (선택)
```

### Phase 6: 다국어 (선택, 30분/언어)

```
1. /jp, /en 등 폴더 생성
2. index.html 복사 → 번역
3. CSS/JS는 공유 (../ 경로)
4. GA4에 language 파라미터 추가
```

### 성공/실패 판단 프레임워크

```
✅ 성공 (Go → MVP 개발):
   - 랜딩 → CTA 전환율 ≥ 5%
   - 스크롤 75% 도달률 ≥ 40%
   - 평균 체류시간 ≥ 60초
   - 모수 ≥ 300명

⚠️ 보류 (Pivot → 메시지/가격 수정):
   - 랜딩 → CTA 전환율 2~5%
   - 특정 세그먼트만 반응 (JP만 높고 KR 낮음 등)
   - → 문구, 가격, 타겟 조정 후 재테스트

❌ 실패 (Kill → 아이디어 폐기):
   - 랜딩 → CTA 전환율 < 2%
   - 스크롤 75% 도달률 < 20%
   - 모든 세그먼트에서 저조
   - → 완전히 다른 아이디어로 피봇
```

### GA4에서 확인하는 방법

1. **페이지뷰**
   - 보고서 → 참여도 → 페이지 및 화면

2. **이벤트 (CTA 클릭 등)**
   - 보고서 → 참여도 → 이벤트
   - `reserve_click` 수 확인

3. **전환율 계산**
   ```
   CTR = reserve_click ÷ page_view × 100
   ```

4. **언어별 비교**
   - 보고서 → 사용자 속성 → 언어
   - 또는 페이지 경로로 필터 (`/` vs `/jp/`)

## 💡 자주 묻는 질문

### Q: 결제 기능 없이 어떻게 검증하나요?
A: CTA 클릭 수 = 구매 의향자 수. 실제 결제 전환율(보통 10-30%)을 곱하면 예상 매출.

### Q: 광고 없이 트래픽은?
A: 인스타 스토리, 카카오톡 공유, 현장 QR 코드 등 오가닉으로 시작.

### Q: 얼마나 테스트해야 하나요?
A: 최소 100 페이지뷰, 권장 500+. 1-2주 운영.

### Q: 실패 기준은?
A: CTA 클릭률 1% 미만이면 메시지/타겟 재검토.
