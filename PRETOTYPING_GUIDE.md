## 📋 필요한 정보 체크리스트

아이디어를 전달할 때 아래 정보를 함께 제공하면 빠르게 진행됩니다.

### 필수 정보
- [ ] **서비스명** (영문/한글)
- [ ] **핵심 가치 (한 문장)** - 예: "손은 가볍게, 성수는 알차게"
- [ ] **타겟 페르소나** - 누가 쓸 것인가?
- [ ] **핵심 액션** - 유저가 뭘 해야 하는가? (예약, 신청, 구매 등)
- [ ] **가격 정책** - 무료/유료, 가격대
- [ ] **위치/장소** (해당 시) - 오프라인 서비스면 주소

### 선택 정보
- [ ] **경쟁사/레퍼런스** - 디자인 참고할 사이트
- [ ] **브랜드 컬러** - 없으면 제가 제안
- [ ] **로고 이미지** - 없으면 나중에 추가
- [ ] **다국어 지원** - 한국어 외 필요한 언어

## 🛠 제작 프로세스

### Phase 1: 기본 구조 (30분)

```
1. 프로젝트 폴더 생성
2. index.html / styles.css / script.js 기본 구조
3. 핵심 섹션 배치
   - 헤더 (로고, 태그라인)
   - 히어로/지도 섹션
   - 옵션 선택 (사이즈, 시간, 플랜 등)
   - 요약 & CTA 버튼
   - 푸터
```

### Phase 2: 디자인 & UX (1시간)

```
1. 브랜드 컬러 적용
2. 모바일 최적화 (타겟이 주로 모바일)
3. 터치 피드백 & 마이크로 인터랙션
4. 폰트 선택 (Pretendard 기본)
```

### Phase 3: 프리토타이핑 로직 (30분)

```
1. CTA 버튼 클릭 시:
   - 1번째 클릭 → "일시적 오류" 모달
   - 2번째 클릭 → 503 에러 페이지
2. 실제 동작 없음 (결제, DB 없음)
```

### Phase 4: 그로스 해킹 설정 (20분)

```
1. Google Analytics 4 연동
2. 이벤트 트래킹 추가
   - page_view (페이지 접속)
   - scroll_depth (스크롤 깊이 25/50/75/100%)
   - button_click (버튼별 클릭)
   - reserve_click (CTA 클릭)
   - session_end (이탈 시 체류시간)
```

### Phase 5: 배포 (10분)

```
1. GitHub 저장소 생성 (Public)
2. git init → commit → push
3. Vercel 연동 (GitHub 로그인 → Import → Deploy)
4. 커스텀 도메인 설정 (선택)
```

### Phase 6: 다국어 (선택, 30분/언어)

```
1. /jp, /en 등 폴더 생성
2. index.html 복사 → 번역
3. CSS/JS는 공유 (../ 경로)
4. GA4에 language 파라미터 추가
```

- 스트림 이름: `[서비스명] Web`

## 💡 자주 묻는 질문

### Q: 결제 기능 없이 어떻게 검증하나요?
A: CTA 클릭 수 = 구매 의향자 수. 실제 결제 전환율(보통 10-30%)을 곱하면 예상 매출.

### Q: 광고 없이 트래픽은?
A: 인스타 스토리, 카카오톡 공유, 현장 QR 코드 등 오가닉으로 시작.

### Q: 얼마나 테스트해야 하나요?
A: 최소 100 페이지뷰, 권장 500+. 1-2주 운영.

### Q: 실패 기준은?
A: CTA 클릭률 1% 미만이면 메시지/타겟 재검토.
